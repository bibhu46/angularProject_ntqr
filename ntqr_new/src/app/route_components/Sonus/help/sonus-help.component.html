GSX

The GSX is the core media gateway component of the Sonus platform. It is responsible for terminating all TDM circuits and converting them to IP. It carries all actual voice bearer traffic as well as the associated IP signalling protocols. It also terminates the SS7 signalling for the TDM circuits.

GSX chassis deployed in COLT's network are sub-divided into two distinct categories:

  • Media Gateway GSX chassis deployed in all 6 POPs handle circuit-to-packet and packet-to-circuit traffic and conversion.
  • Network Border Switch GSX chassis with no circuit capacity handle IP-to-IP traffic only and are deployed in 4 high capacity POPs (Frankfurt, London, Paris and Zurich)

PSX

The PSX is the core routing engine of the Sonus platform. It is responsible for all call-routing logic and decisions.

Active Call Count

The active call count represents the number of stable and transient calls. An anomalous drop in active calls can indicate an overload event.

PSX Dip Retries and Failures

Possible causes for Retries and Failures are GSX congestion, PSX capacity shortage, and TCAP timeout. Depending on the cause GSX, or PSX capacities or the timer values should be reviewed.

Machine Congestion Rejects -NRM

Machine Congestion Rejects indicates the number of calls shed as a result of machine overload. The reasons can include processor or memory shortage due to high traffic volume, or excessive call terminations due to an internal or external failure.

Server Congestion Rejects - DS

Server Congestion Rejects indicates the number of calls shed as a result of server overload. The reasons can include processor or memory shortage on a server due to high traffic volume, or excessive call terminations due to an internal or external failure focused to a particular server.

Memory Utilization

If the reading is consistently above the threshold in the busy hour then box capacity should be reviewed.

CPU Utilization

If the reading is consistently above the threshold in the busy hour then box capacity should be reviewed.

VLAN Numbers

This command shows a summary of all interfaces on a GSX or NBS chassis. From the output the number of configured VLANs can be deduced.

External Timeout Count

This is an indication of external query timeouts. The reasons include SS7 network issues, SCP failure, or PSX processor shortage.

Internal Service Time

Internal service time reflects the time that it takes for PSX to answer the policy inquiries. A spike in service times may be an indication of a performance bottleneck.

Dip Rate

Dip rate is useful when analyzing a performance issue.

Disk Busy Percentage

Spikes due to maintenance activities (such as CDR file processing) should be monitored closely against the threshold.

Read-Write Rate

When Read-Write Rate is above 250 (for local disk) and 1500 (for RAID) CDR/log loss is likely to occur. Spikes due to maintenance activities (such as CDR file processing) should be monitored closely against the thresholds.

Waiting Transactions

A waiting transactions value above zero is an indication for I/O bottleneck. If the threshold is crossed due to GSX logging capacity augmentation should be considered.

Disk Space Utilization

If the system partitions (such as /, /var) or the ones that are used by Sonus applications cross the threshold older or unwanted files should be cleaned up

MNS - Management Network Server

Communicate with all other Sonus devices on the management network, such as the PSX, SGX, EMS, and DSI.Load software from the NFS server at boot-up and direct download files to other server modules in the chassis.

Monitor system power, cooling, and the operational state of all other modules. Write events to log files on the NFS server, including call detail records (CDRs).Generate SNMP alarms and send them to the EMS server.

CNS - Circuit Network Server

Provides time-division multiplexed (TDM) connections to the PSTN.

PNS - Packet Network Server

Provides Ethernet connections to a packet network for carrying packetized voice. Ports can carry both call signaling packets and voice traffic
